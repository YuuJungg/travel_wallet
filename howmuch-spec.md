# 얼마지 — 여행 지출 트래커 기능 명세서

> **태그라인:** 이거 얼마지? 나 얼마 남았지?
> **이번 빌드 목표:** 다음 후쿠오카 여행에서 나(유정) 혼자 쓰는, 완전 오프라인으로 도는 현금 지출 트래커.

이 문서는 Claude Code가 그대로 읽고 구현할 수 있도록 작성됨. 기능·데이터 모델·계산식이 우선이고, 비주얼은 마지막 섹션 참고.

---

## 0. 핵심 제약 (반드시 지킬 것)

1. **단일 HTML 파일** — vanilla JavaScript, 외부 빌드 도구 없음.
2. **localStorage 영속화** — 새로고침/앱 재실행해도 데이터 유지.
3. **완전 오프라인 동작** — 실시간 환율 API, 외부 폰트 CDN 의존 등 **인터넷 없으면 깨지는 요소 금지**. (폰트는 못 불러와도 기본 폰트로 무난하게 보이게 fallback 지정)
4. **통화 하드코딩 금지** — `¥`, `엔`, `100엔당` 같은 표기를 코드에 직접 박지 말고 통화 설정 객체에서 읽어 쓸 것. 지금은 JPY 단일이지만 나중에 다른 나라 통화로 교체/확장 가능해야 함.
5. **사람(누가 썼나) 하드코딩 금지** — "나"·"준보"를 코드에 박지 말고 사람 목록을 데이터로 둘 것. 나중에 다른 사람과의 여행에도 재사용 가능하게.

---

## 1. 개요

여행 중 현금(엔화)을 한 명이 들고 다니며 모든 지출을 대신 결제하는 상황을 가정한다. 앱은 다음을 한다.

- 가진 현금(충전한 엔화)과 쓴 돈을 추적해 **남은 잔액**을 항상 보여준다.
- 각 지출을 **상품명 / 분류(태그) / 금액(엔) / 누가**로 기록한다.
- 환전 시점마다 **내가 지정한 환율**을 기록해 모든 금액을 **한화로 함께** 보여준다.
- 여행이 끝나면 분류를 합산해 **정산**(누가 나에게 얼마 줄지)을 보여준다.

## 2. 데이터 모델

localStorage에 JSON으로 저장. 최상위 키 예시: `eolmaji.v1`.

```js
{
  // 통화 설정 — 하드코딩 대신 여기서 읽어 쓴다
  "currency": {
    "code": "JPY",
    "symbol": "¥",
    "name": "엔",
    "decimals": 0          // 엔: 소수점 없음
  },
  "homeCurrency": {
    "code": "KRW",
    "symbol": "₩",
    "name": "원",
    "decimals": 0          // 원: 소수점 없음
  },

  // 사람 목록 — 하드코딩 금지. owner = 현금을 들고 결제하는 사람(=나)
  "people": [
    { "id": "p_me",   "name": "나",   "isOwner": true  },
    { "id": "p_2",    "name": "준보", "isOwner": false }
  ],

  // 분류 태그 — 사용자가 생성해 재사용/삭제 (노션 스타일)
  "tags": [
    { "id": "t_1", "name": "식비",   "color": "#d6452c" },
    { "id": "t_2", "name": "교통",   "color": "#3c7a89" }
  ],

  // 충전(환전) 이벤트 — 환전할 때마다 1건씩
  "charges": [
    {
      "id": "c_1",
      "date": "2026-06-22",
      "amountForeign": 50000,   // 받은 엔화
      "paidHome": 470000        // 그 엔화를 사느라 낸 원화 (총액)
      // 이 건의 환율 = paidHome / amountForeign
    }
  ],

  // 지출 기록
  "expenses": [
    {
      "id": "e_1",
      "date": "2026-06-23",
      "item": "이치란 라멘",
      "amountForeign": 1200,    // 금액(엔)
      "tagId": "t_1",
      "who": "p_2"              // people의 id  또는  "SHARED"(공동)
    }
  ]
}
```

**중요 규칙**

- `who` 값은 사람 `id` 또는 특수값 `"SHARED"`(공동) 중 하나.
- 한화 금액은 **저장하지 않고** 가중평균환율로 매번 계산한다(아래 §4). 환율을 나중에 조정해도 전체가 일관되게 따라오도록.
- 사람 입력란은 owner(나) / 다른 사람들 / 공동(SHARED) 순으로 노출.

## 3. 화면 구성

탭 또는 하단 네비게이션 5개 + 설정.

### 3-1. 홈 (잔액)
- **남은 엔화**를 가장 크게 표시 + 그 아래 한화 환산(작게).
- 보조 요약: 총 충전 엔 / 총 지출 엔 / 적용 중인 평균 환율(100엔 = N원 형태로도 표기).
- **빠른 환산기**: 엔 금액을 입력하면 한화로 즉시 환산해 보여줌(평균 환율 사용). 가게에서 "이거 사면 한화 얼마지?" 확인용. (오프라인, 내 환율 기준)
- "지출 추가" 빠른 버튼.

### 3-2. 지출 추가
입력 필드:
- 상품명 (텍스트)
- 금액 (엔, 정수) — 입력 중 한화 환산값을 옆에 실시간 표시
- 분류 — 기존 태그 칩에서 고르거나, 새 태그 입력 시 생성되어 목록에 저장됨
- 누가 — 나 / 준보 / 공동 중 선택
- 날짜 (기본값: 오늘)

### 3-3. 지출 내역
- **날짜별 그룹**으로 묶어 표시.
- 각 항목: 상품명 · 태그 · 금액(엔) · 한화 · 누가.
- 각 항목 **수정 / 삭제** 가능.

### 3-4. 충전(환전)
- 충전 추가: 받은 엔화 + 낸 원화 입력 → 그 건의 환율 자동 산출.
  - (대안 입력) 환율을 직접 넣고 싶을 땐 "환율 직접 입력" 토글 → 환율 입력 시 `paidHome = amountForeign × 환율`로 환산해 저장.
- 충전 내역 리스트(날짜·엔화·그때 환율), 수정/삭제.

### 3-5. 정산
- 사람별 지출 합계 + 공동 합계 표시.
- **결과**: owner를 제외한 각 사람에 대해 "○○가 나에게 줄 돈"을 계산해 크게 표시(§4-3).
- 엔·한화 동시 표기.

### 3-6. 설정
- 사람 관리(추가/이름변경/삭제, owner 지정).
- 태그 관리(이름변경/삭제).
- 통화 설정(현재 JPY, 향후 교체 가능하게 노출).
- 데이터 초기화.

---

## 4. 계산 로직 (정확히 이대로)

### 4-1. 환율 (가중평균)
```
평균환율(원/엔) = Σ(charges.paidHome) / Σ(charges.amountForeign)
```
- 충전이 0건이면 환율 없음 → 한화 표시는 "—"로 두고, 충전 추가를 유도.
- 100엔 기준 표기 = 평균환율 × 100.
- 충전이 여러 번이고 환율이 다르면 자연스럽게 가중평균이 됨.

### 4-2. 잔액
```
총충전엔   = Σ(charges.amountForeign)
총지출엔   = Σ(expenses.amountForeign)
남은엔     = 총충전엔 − 총지출엔
남은한화   = round(남은엔 × 평균환율)        // 원은 정수 반올림
```
- 남은엔이 음수가 될 수 있음(과지출) → 빨간색 경고 표시.

### 4-3. 정산
- owner = `isOwner: true`인 사람(나). owner의 개인 지출은 정산 대상 아님.
- 공동(SHARED)은 전체 인원수(`people.length`)로 나눠 분담.

owner가 아닌 각 사람 P에 대해:
```
P의_개인지출 = Σ(expenses where who == P.id).amountForeign
공동총액      = Σ(expenses where who == "SHARED").amountForeign
P의_공동분담  = 공동총액 / people.length

P가_나에게_줄돈(엔) = P의_개인지출 + P의_공동분담
P가_나에게_줄돈(원) = round(위 × 평균환율)
```
- 현재 여행(나 + 준보 = 2명) 기준이면 `공동분담 = 공동총액 / 2`가 되어,
  **준보가 줄 돈 = 준보 개인지출 + 공동 ÷ 2**. 의도대로 동작함.

### 4-4. 한화 환산 (공통)
- 어떤 엔 금액이든 한화 = `round(엔 × 평균환율)`.
- 표기는 통화 설정의 `symbol`/`decimals`를 사용(하드코딩 금지).

## 5. 동작 디테일 / 엣지 케이스

- 충전 0건일 때: 잔액·정산의 한화는 "—", 환율 안내 문구 노출.
- 분모 0 방지: 평균환율 계산 시 `총충전엔 == 0`이면 환율 미정 처리.
- 태그 생성: 같은 이름 중복 생성 막기(있으면 기존 것 사용).
- 태그 삭제: 그 태그를 쓰던 지출은 "분류 없음"으로 남기고 깨지지 않게.
- 사람 삭제: 그 사람에게 달린 지출 처리 방식(차단 or 재할당)을 확인 모달로.
- 모든 금액 입력은 정수(엔/원 모두 decimals=0). 숫자 외 입력 방지.
- 수정/삭제는 확인 후 즉시 localStorage 반영.

## 6. 비주얼 / 브랜드 (얼마지 아이덴티티)

기존 얼마지 정체성 유지. 따뜻한 아날로그·영수증 느낌.

- **이름:** 얼마지 / **태그라인:** "이거 얼마지? 나 얼마 남았지?"
- **폰트:** Zen Maru Gothic + Gowun Dodum (오프라인 fallback 필수 — 못 불러오면 시스템 기본 고딕)
- **컬러:** 레드 액센트 `#d6452c`, 크림 배경 `#faf6ef`
- **무드:** 영수증 스타일 카드, 잔액을 영수증 합계처럼 강조.

## 7. 향후 확장 (지금 구현 X, 막지만 말 것)

- **다통화:** `currency` 객체 교체만으로 다른 나라 통화 대응. 추후 여행별로 통화 선택.
- **다인원:** `people` 배열 확장 시 공동 분담이 `/ people.length`로 자동 대응되도록 이미 설계됨.
- 여행(trip) 단위로 데이터 분리(여러 여행 저장).

---

## 8. 구현 우선순위

1. 데이터 모델 + localStorage 입출력
2. 충전 입력 → 평균환율 계산
3. 지출 추가 / 내역 / 수정·삭제
4. 홈 잔액 + 한화 환산 + 빠른 환산기
5. 태그·사람 관리
6. 정산 화면
7. 비주얼 마감
